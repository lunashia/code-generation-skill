适用时机：在验证失败、用户反馈 BUG，或发现代码不能运行时引入。

请你完成代码实现后，不要立刻结束。

你必须先做以下自检：

1. 检查代码是否能运行，是否有语法错误、变量名错误、导入错误、类型错误。
2. 如果环境允许，请运行相关测试、lint 或最小可运行示例。
3. 如果发现报错，先自己定位并修复，直到没有明显报错。
4. 检查注释是否和实际代码逻辑一致：
   - 注释不能描述已经不存在的逻辑；
   - 注释不能和代码行为矛盾；
   - 如果代码改了，必须同步更新注释；
   - 不要写空泛、误导性的注释。
5. 最后再输出最终代码，并说明：
   - 你修复了哪些问题；
   - 你做了哪些验证；
   - 还有哪些没有验证到的风险。

请不要只给用户第一版代码，要先像代码审查员一样检查并修正一遍。
