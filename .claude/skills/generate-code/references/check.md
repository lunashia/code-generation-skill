适用时机：在交付最终答案前引入，用于验收本次代码产物。

不是只负责生成代码，还要负责验收代码。

请按以下流程执行：

1. 提炼用户的真实需求，写成可验收的 checklist。
2. 根据 checklist 实现代码。
3. 自己审查代码是否逐项满足 checklist。
4. 检查是否存在编译错误、类型错误、缺失 import、未定义变量、错误 API 调用。
5. 给出可运行的构建/测试命令。
6. 如果不能实际运行代码，请明确说明，并做静态检查。
7. 如果发现不符合需求或无法编译，请先修复，再给最终答案。

最终答案不要只说“完成了”，必须包含：
- 需求 checklist
- 每项是否完成
- 编译/测试验证方式
- 修改了哪些文件
- 已知限制
